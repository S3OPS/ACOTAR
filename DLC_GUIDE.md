# ACOTAR DLC Content Guide

## Overview

The ACOTAR Fantasy RPG uses a base game + DLC model based on the book series:

| Content | Book | Status | Requirements |
|---------|------|--------|--------------|
| **Base Game** | A Court of Thorns and Roses | Included | None |
| **DLC 1** | A Court of Mist and Fury | Purchasable | Base Game |
| **DLC 2** | A Court of Wings and Ruin | Purchasable | Base Game + DLC 1 |

## Technical Implementation

### DLCManager

The `DLCManager` class handles all DLC-related functionality:

```csharp
// Check if DLC is owned
bool owned = DLCManager.Instance.IsDLCOwned(DLCPackage.ACOMAF_MistAndFury);

// Activate/purchase DLC
DLCManager.Instance.ActivateDLC(DLCPackage.ACOMAF_MistAndFury);

// Install DLC content
DLCManager.Instance.InstallDLC(DLCPackage.ACOMAF_MistAndFury, questDictionary);

// Check if content is accessible
bool canAccess = DLCManager.Instance.CanAccessDLCContent(DLCPackage.ACOWAR_WingsAndRuin, storyManager);

// Display DLC status
DLCManager.Instance.DisplayDLCStatus();
```

### Quest Gating

Quests are automatically gated based on DLC ownership:
- Base game quests (main_*, side_001-009, companion_001-002) always available
- Book 2 quests (book2_*, side_010-015, companion_003-005) require DLC 1
- Book 3 quests (book3_*, side_016+, companion_006+) require DLC 2

### Story Progression

When completing the base game (Book1_Aftermath arc):
1. Player sees "BASE GAME COMPLETE" message
2. System checks for DLC 1 ownership
3. If owned: Continues to Book2_NightCourt arc
4. If not owned: Prompts player to purchase DLC
